## 2025-02-12 - [Testing] Add test coverage for ExportController exception handling
**Learning:** Testing exception handling logic in ASP.NET Core controllers often requires mocking the underlying service to throw expected exceptions. When the service is not behind an interface but is a concrete class injected directly, `Moq` requires the target methods to be `virtual` so it can create a proxy and override their behavior. Additionally, creating a minimal `DbContextOptionsBuilder` with `UseInMemoryDatabase` is a good way to satisfy `DbContext` dependencies when instantiating the real service class within tests, even if the primary goal is mocking.
**Action:** Made all export methods in `ExportService` `virtual` to enable mocking, and added a comprehensive test suite in `ExportControllerTests.cs` covering the `catch` blocks for all export endpoints.
